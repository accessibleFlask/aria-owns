What are WCAG and ARIA?

WCAG

WCAG is an acronym for Web Content Accessibility Guidelines. These are guidelines to improve web accessibility. They're divided into 3 levels:

  1. Level A - basic accessibility
  2. Level AA - addresses the biggest barriers to web accessibility for most users
  3. Level AAA - the highest level of accessibility

The WCAG guidelines are based on the POUR principle, which states that websites should be:

Users should be able to percieve all content on your website, regardless of ability
Users should be able to navigate and interact with your website, regardless of how (mouse, keyboard, etc)
Users should be able to comprehend the content on your site. Instructions should be clear, concise, and easy to understand. Interactions should be predictable and consistent
Your site should be standards-compliant and reliable. Cross-browser testing is encouraged as well.

ARIA

ARIA is the acronym for Accessible Rich Internet Applications. This is a set of attributes that make web content more accessible to users employing assistive technologies. We'll get into some basic ARIA attributes in this talk, but the basics of ARIA are that ARIA attributes can be added to markup to define things like roles (what function a specific portion of the page has, like headers or navigation), or descriptions, or error functionality.
